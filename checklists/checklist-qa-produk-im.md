# Checklist QA Produk IM

## Ringkasan

Checklist ini dipakai untuk memeriksa kualitas produk IM sebelum dirilis ke Manajer Operasi/Kepala Operasi atau disebarluaskan. Output IM yang buruk bisa menyebabkan keputusan operasi yang salah. QA adalah harga mati.

## Data Check

- [ ] Angka-angka sudah cocok dengan sumber - tidak ada salah ketik
- [ ] Total dan sub-total sudah diverifikasi (tidak asal jumlah)
- [ ] Satuan sudah benar (orang, KK, RT, liter, dll)
- [ ] Tanggal dan periode data sudah sesuai
- [ ] Tidak ada data duplikat
- [ ] Pembulatan angka konsisten

## Format Check

- [ ] Logo PMI dan标识 (logo/nama) IM Cell PMI tercantum
- [ ] Judul produk jelas dan sesuai isi
- [ ] Tanggal publikasi/rilis tercantum
- [ ] Nomor halaman (jika > 1 halaman)
- [ ] Tabel punya header yang jelas
- [ ] Grafik punya judul dan sumbu yang diberi label
- [ ] Tidak ada teks terpotong atau tabel terbelah
- [ ] Margin dan tata letak rapi - siap cetak
- [ ] Disclaimer/classification (INTERNAL/RAPATTERBATAS/PUBLIK) tercantum

## Source Check

- [ ] Setiap angka/data punya sumber yang jelas
- [ ] Kredibilitas sumber sudah dinilai (langsung/tidak langsung, diverifikasi?)
- [ ] Tanggal pengambilan data tercatat
- [ ] Metode pengumpulan data tercatat (survey, wawancara, data sekunder)
- [ ] Jika data dari pihak ketiga, izin pakai sudah didapat

## Before Release Check

- [ ] Semua cek di atas sudah dilakukan
- [ ] Produk sudah diperiksa oleh pendamping / IM Senior
- [ ] Produk sudah disimpan di folder yang benar dengan nama file sesuai konvensi
- [ ] Produk sudah di-backup (cloud/share drive)
- [ ] Penerima produk sudah ditentukan (Manajer Operasi/Kepala Operasi, sektor terkait, dll)
- [ ] Produk dikirim tepat waktu sesuai deadline
