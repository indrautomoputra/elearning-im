# Setup dan Orientasi Sistem

Persiapan sistem dan alat kerja IM Cell PMI agar kamu bisa langsung produktif di posko.

---

## Kapan Topik Ini Dipakai di Posko

Topik ini untuk hari pertama kamu masuk posko. Sebelum mulai mengelola data, pastikan semua akses sistem sudah siap: email, Microsoft 365, GitHub, folder bersama, dan alat komunikasi.

---

## Yang Harus Kamu Bisa Lakukan

- [ ] Akses dan navigasi MS Teams dan SharePoint
- [ ] Cari file di struktur folder IM Cell PMI
- [ ] Buat dan pindahkan kartu di GitHub Kanban
- [ ] Gunakan format penamaan file standar

---

## Gambaran Singkat

IM Cell PMI menggunakan beberapa sistem sekaligus. MS Teams untuk komunikasi harian, SharePoint untuk penyimpanan dokumen, dan GitHub Kanban untuk melacak tugas.

Kuncinya: konsistensi. Nama file, struktur folder, dan cara update tugas harus seragam biar semua orang bisa kerja tanpa kebingungan.

---

## Langkah Kerja

### Onboarding Sistem

1. Minta akses ke pendamping IM untuk semua sistem berikut:
  - Email PMI
  - Microsoft 365 (Teams, SharePoint, Excel Online)
  - GitHub organisasi
  - KoboToolbox (jika bertugas kelola survei)
2. Gabung ke channel MS Teams IM Cell PMI dan grup WhatsApp
3. Buka folder SharePoint dan pelajari struktur foldernya
4. Cek GitHub board - lihat tugas yang ada di kolom To Do

### Penamaan File

Gunakan format: `YYYY-MM-DD_Tipe_Deskripsi_v{N}.ext`

Contoh: `2024-01-15_SitRep_Harian_v1.xlsx`

### Navigasi MS Teams

- `Ctrl + E` - cari pesan
- `Ctrl + 1-9` - pindah channel cepat
- `/call` - panggilan cepat

---

## Contoh Lapangan

Hari pertama kamu di posko. Pendamping IM memberi akses ke SharePoint. Kamu lihat folder `IM-Cell/01_Data/Incoming/` - isinya sudah ada 5 file dari sektor kesehatan, shelter, dan WASH. Kamu buka file, lihat nama file-nya konsisten, langsung paham kapan data masuk dan dari mana.

Di GitHub, kamu ambil kartu "Update 4W Matrix" dari kolom To Do, pindahkan ke In Progress, dan mulai kerja.

---

## Tugas Praktik OJT

1. Minta pendamping membuatkan akun/akses ke semua sistem yang diperlukan
2. Login ke MS Teams dan eksplorasi setiap channel IM Cell PMI
3. Buka folder SharePoint dan catat struktur foldernya
4. Buat 3 file contoh dengan nama standar di folder latihan
5. Buat 1 issue di GitHub dan pindahkan kartu dari To Do ke Done

---

## Output Minimum

- Semua akses sistem sudah aktif dan bisa dipakai
- 1 file contoh dengan format nama standar
- 1 kartu GitHub yang pernah dipindahkan di board

---

## Checklist Cepat

- [ ] Email dan M365 sudah aktif
- [ ] MS Teams channel sudah diikuti
- [ ] Folder SharePoint bisa diakses
- [ ] GitHub board bisa dibuka
- [ ] Format penamaan file sudah dipahami

---

## Kesalahan Umum

| Kesalahan | Dampak | Pencegahan |
|-----------|--------|------------|
| Upload file ke folder sembarangan | Dokumen sulit ditemukan | Cek struktur folder dulu sebelum upload |
| Nama file tidak konsisten | Orang lain bingung, file duplikat | Gunakan format standar |
| Tidak ada backup offline | File hilang jika server down | Simpan salinan lokal setiap hari |

---

## Kapan Harus Eskalasi

Eskalasi ke pendamping jika:
- Akses sistem tidak diberikan setelah 1x24 jam
- File di folder Incoming tiba-tiba berubah tanpa jejak
- Ada konflik nama file karena dua orang edit file sama
- Aplikasi tidak bisa diinstal karena kebijakan IT

## Catatan untuk Pendamping

Relawan baru bisa kewalahan dengan banyaknya sistem. Prioritaskan: akses email dan Teams dulu, baru SharePoint dan GitHub.

Buat panduan cetak 1 halaman - langkah-langkah login dan navigasi dasar. Tempel di meja posko.

---

## Ringkasan Cepat

- Pastikan akses semua sistem siap di hari pertama
- Gunakan format nama file standar
- Struktur folder: Incoming (mentah) > Cleaned (siap olah) > Archive (lama)
- GitHub: To Do > In Progress > Review > Done
- Jangan edit file di folder Incoming - salin dulu ke Cleaned
