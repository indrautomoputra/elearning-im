# Paket Data Pendukung RenOps

Menyusun paket data yang menjadi dasar Rencana Operasi (RenOps).

---

## Kapan Topik Ini Dipakai di Posko

Topik ini dipakai saat perencanaan operasi - biasanya setelah PIN dan analisis gap selesai. Kamu akan menyusun paket data yang berisi profil wilayah, data dampak, analisis, dan rekomendasi untuk RenOps.

---

## Yang Harus Kamu Bisa Lakukan

- [ ] Susun paket data RenOps lengkap (5 komponen)
- [ ] Cek kualitas paket data dengan checklist standar
- [ ] Buat visualisasi data yang tepat sesuai tujuan dan audiens

---

## Gambaran Singkat

RenOps adalah dokumen perencanaan yang menjabarkan strategi, sumber daya, dan kegiatan operasi. IM Cell PMI menyediakan data dan analisis yang menjadi dasar RenOps.

Paket data RenOps terdiri dari 5 komponen:
1. **Profil Wilayah** - demografi, baseline, peta administrasi
2. **Data Dampak** - korban, pengungsi, kerusakan
3. **Kapasitas Respons** - sumber daya, 4W, inventaris
4. **Analisis** - PIN, gap analysis, prioritas
5. **Rekomendasi** - target geografis, sektoral, timeline

---

## Langkah Kerja

### Menyusun Paket Data

1. **Kumpulkan profil wilayah**: peta administrasi, demografi, data baseline
2. **Kumpulkan data dampak**: korban, pengungsi, kerusakan, aksesibilitas
3. **Kumpulkan data kapasitas**: sumber daya PMI, 4W, inventaris
4. **Buat analisis**: hitung PIN, gap analysis, prioritas intervensi
5. **Tulis rekomendasi**: target geografis, target sektoral, timeline

### Cek Kualitas

Sebelum serah, cek:
- Semua file pakai format nama konsisten
- Ada README di folder utama
- Data sudah dibersihkan - tidak ada missing values kritis
- Sumber dan tanggal tercantum
- PIN dan gap analysis sudah direview

### Visualisasi

Tentukan tujuan visual: perbandingan? - bar chart. Tren? - line chart. Geografis? - peta.

Kenali audiens: pengambil keputusan butuh ringkasan, tim teknis butuh detail. Satu visual maksimal 5 warna.

## Kesalahan Umum

| Kesalahan | Dampak | Pencegahan |
|-----------|--------|------------|
| Paket data tidak ada README | Orang lain bingung isinya | Selalu sertakan README |
| Data tanpa tanggal | Tidak tahu relevansi data | Cantumkan tanggal di setiap file |
| Visual terlalu ramai | Pembaca tidak paham | Maksimal 5 warna, info penting di kiri atas |
| Tidak ada peer review | Kesalahan lolos ke Manajer Operasi | Minta review pendamping sebelum kirim |

---

## Kapan Harus Eskalasi

Eskalasi ke pendamping jika:
- Data dasar (profil wilayah, demografi) tidak tersedia
- Ada konflik data antar sumber yang tidak bisa diselesaikan
- Paket data diminta lebih cepat dari waktu yang realistis
- Kamu diminta merekomendasikan prioritas tanpa data gap yang cukup

## Catatan untuk Pendamping

Paket data RenOps adalah output krusial IM Cell PMI. Kualitasnya menentukan kualitas perencanaan operasi. Ajarkan relawan untuk selalu cek: "Apakah paket data ini bisa dipakai orang lain tanpa penjelasan lisan?"

Latih relawan melakukan peer review - saling cek paket data sebelum diserahkan.

## Ringkasan Cepat

- Paket data RenOps = profil + dampak + kapasitas + analisis + rekomendasi
- Folder harus punya README - jangan buat orang lain menebak
- Data tanpa tanggal = tidak berguna - cantumkan selalu
- Peer review sebelum serah - mata segar lihat kesalahan yang terlewat
- Visual yang baik = 5 detik paham tanpa penjelasan
